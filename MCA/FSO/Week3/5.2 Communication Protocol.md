### Why Protocols Are Needed
- Simply having a **wired or wireless physical connection** between end devices is **not enough** to enable communication.

	- For communication to occur, **~={green}devices must know "how" to communicate.=~**

### Definition of Protocols
- **Protocols** are **~={green}rules that govern communication.=~**

	- Communication, can be **face-to-face** or **over a network**, and **it is governed by a protocol.**
	- Protocols **are of specific type** and i**~={green}t is based on the communication method that is  occurring.=~**

#### Real-World Analogy: Face-to-Face Communication

~={yellow}**Before two people can communicate**=~ face-to-face, they must ~={green}**agree on how to communicate**:=~

1. **Agree on the Language** - If using voice, ~={cyan}they must first agree on the language=~
2. **Format of the Message** - ~={cyan}They must format messages in an understandable way=~
    - Example: Poor sentence structure in English makes the message easily misunderstood
    - Good structure ensures the message is clear and understood

Similarly, **network protocols work the same way** for devices communicating over networks.

### What are Network Protocols ? 

~={blue}**Network protocols**=~ ~={green}provide the means for computer communication=~ by 
- dictating message encoding, 
- formatting, 
- encapsulation, 
- size, timing, 
- delivery options.

They define **~={orange}common formats=~** and **~={orange}rules for exchanging messages=~** between devices.

# TCP/IP Model - Quick Reference Guide

## Overview

A **~={blue}four-layer networking framework=~** that standardizes **~={green}how data is communicated=~** **~={yellow}between devices over networks.=~** 
- Simpler and more practical than the OSI model.

---

## Layer 1: Application Layer

- The **top layer of the TCP/IP** model, **closest to the user**, where **applications** like ~={yellow}web browsers, email clients, and file-sharing tools=~ **~={green}interact with the network.=~** 
- **~={orange}It provides an interface=~** between **~={blue}user software and the lower layers=~** enabling seamless communication over the network.=~**

**Key Functions**: **~={cyan}Data formatting, encryption, session management=~**

**Protocols**:

| Category          | Protocol | Purpose                                            |
| ----------------- | -------- | -------------------------------------------------- |
| **Name System**   | DNS      | Translates domain names to IP addresses            |
| **Host Config**   | DHCPv4   | Assigns IPv4 addresses automatically               |
|                   | DHCPv6   | Assigns IPv6 addresses automatically               |
|                   | SLAAC    | IPv6 self-assignment without DHCP                  |
| **Email**         | SMTP     | Sends email between clients and servers            |
|                   | POP3     | Retrieves and downloads email to client            |
|                   | IMAP     | Accesses email on server                           |
| **File Transfer** | FTP      | Reliable file transfer (connection-oriented)       |
|                   | SFTP     | Secure file transfer (encrypted with SSH)          |
|                   | TFTP     | Fast file transfer (connectionless, less overhead) |
| **Web**           | HTTP     | Exchanges multimedia on the Web                    |
|                   | HTTPS    | Encrypted HTTP for secure communication            |
|                   | REST     | Web services using APIs and HTTP requests          |

---

## Layer 2: Transport Layer

**~={blue}Ensures ~={orange}reliable and efficient delivery of data=~ between devices, ~={yellow}managing segmentation=~, ~={yellow}ordering=~, and ~={yellow}retransmission as needed.=~=~**

**Key Functions**: **~={orange}Segmentation, error handling, flow control, multiplexing=~**

### TCP (Transmission Control Protocol)

- **Reliable**: Confirms data delivery
- **Error Checking**: Detects and retransmits lost data
- **Ordered**: Packets arrive in correct sequence
- **Connection-Oriented**: Establishes connection before sending
- **Use Cases**: Web browsing, file downloads, email, video calls

### UDP (User Datagram Protocol)

- **Fast**: Prioritizes speed over accuracy
- **No Error Checking**: Does not verify data integrity
- **No Retransmission**: Lost data is not resent
- **Connectionless**: No connection setup
- **Use Cases**: Live streaming, online gaming, VoIP, real-time apps

## Layer 3: Internet Layer

- **Responsible** for **~={orange}addressing, packaging, and routing data packets=~** so they **~={green}can travel across networks and reach the correct destination device.=~** 
- It **ensures that data can move between different networks efficiently**.

**Key Functions**: **~={cyan}Logical addressing, packet routing, fragmentation=~**

**Protocols**:

| Category              | Protocol  | Purpose                                                      |
| --------------------- | --------- | ------------------------------------------------------------ |
| **Internet Protocol** | IPv4      | 32-bit addressing; standard IP protocol                      |
|                       | IPv6      | 128-bit addressing; newer IP standard                        |
|                       | NAT       | Translates private IPs to public IPs                         |
| **Messaging**         | ICMPv4    | Error feedback for IPv4 packets                              |
|                       | ICMPv6    | Error feedback for IPv6 packets                              |
|                       | ICMPv6 ND | Address resolution and duplicate detection                   |
| **Routing**           | OSPF      | Open standard interior routing (hierarchical)                |
|                       | EIGRP     | Cisco routing protocol (bandwidth, delay, load, reliability) |
|                       | BGP       | Exterior routing between ISPs                                |

---

## Layer 4: Network Access (Link Layer)

- **Responsible for~={yellow} physically transmitting data over network hardware**=~, including **cables**, **switches**, and **wireless** **connections**. 
- It **~={green}handles how data is formatted=~** so that the **~={cyan}network medium and ensures it reaches the next device=~** on the path.

**Key Functions**: Physical transmission, framing, MAC addressing, error detection

**Protocols**:

|Category|Protocol|Purpose|
|---|---|---|
|**Address Resolution**|ARP|Maps IPv4 addresses to MAC addresses|
|**Data Link**|Ethernet|Wiring and signaling standards|
||WLAN|Wireless signaling (2.4 GHz and 5 GHz)|

---

## Quick Summary

|Layer|Name|Focus|Example Protocols|
|---|---|---|---|
|1|Application|User applications|HTTP, FTP, DNS, SMTP|
|2|Transport|Reliable delivery|TCP, UDP|
|3|Internet|Routing and addressing|IP, ICMP, BGP|

# OSI Model - Quick Reference Guide

## Overview

A **~={green}seven-layer networking framework=~** that **standardizes** **how data is communicated between devices over networks**. 
- More **detailed than the TCP/IP model**, providing a comprehensive view of network communication.

---

## Layer 1: Physical Layer

Handles the physical transmission of raw bits over network hardware.

**Key Functions**: Transmits electrical signals, defines cable types, voltage levels

**Devices**: Cables, hubs, repeaters, network interface cards (NICs)

**Standards**:

- Ethernet cabling (Cat5, Cat6, Cat7)
- Fiber optic cables
- Wi-Fi radio frequencies
- USB connections
- RS-232 serial connections

---

## Layer 2: Data Link Layer

Organizes data into frames and handles communication between adjacent devices on the same network segment.

**Key Functions**: Framing, MAC addressing, error detection, physical addressing

**Protocols**:

|Protocol|Purpose|
|---|---|
|**Ethernet**|Defines wiring and signaling standards; most common LAN technology|
|**WLAN (Wi-Fi)**|Wireless communication on 2.4 GHz and 5 GHz frequencies|
|**PPP**|Point-to-Point Protocol; used for direct connections|
|**Frame Relay**|WAN protocol for packet switching|
|**ATM**|Asynchronous Transfer Mode; high-speed transmission|

**Devices**: Switches, bridges, NICs

**Note**: ARP (Address Resolution Protocol) maps IPv4 to MAC addresses here

---

## Layer 3: Network Layer

Routes data packets across different networks to reach the correct destination.

**Key Functions**: Logical addressing (IP), packet routing, fragmentation, path determination

**Protocols**:

|Category|Protocol|Purpose|
|---|---|---|
|**Internet Protocol**|IPv4|32-bit addressing; standard IP protocol|
||IPv6|128-bit addressing; newer IP standard|
||NAT|Translates private IPs to public IPs|
|**Messaging**|ICMP|Error reporting and diagnostics|
||ICMPv4|Error feedback for IPv4 packets|
||ICMPv6|Error feedback for IPv6 packets|
|**Routing**|OSPF|Link-state interior routing protocol|
||EIGRP|Enhanced interior gateway routing protocol|
||BGP|Exterior gateway routing between ISPs|
||RIP|Routing Information Protocol|
|**Address Resolution**|ARP|Maps IPv4 addresses to MAC addresses|

**Devices**: Routers, layer-3 switches

---

## Layer 4: Transport Layer

Ensures reliable and efficient delivery of data between devices.

**Key Functions**: Segmentation, error handling, flow control, multiplexing using ports

**Protocols**:

|Protocol|Type|Characteristics|Use Cases|
|---|---|---|---|
|**TCP**|Connection-Oriented|Reliable, ordered, error-checked|Web browsing, email, file downloads|
|**UDP**|Connectionless|Fast, no error checking|Live streaming, gaming, VoIP|
|**SCTP**|Hybrid|Reliable, message-oriented|Telecommunications, signaling|

**Port Numbers**: 0-65535

- Well-known ports: 0-1023
- Registered ports: 1024-49151
- Dynamic ports: 49152-65535

---

## Layer 5: Session Layer

Establishes, manages, and terminates communication sessions between applications.

**Key Functions**: Session establishment, maintenance, termination, dialog control

**Protocols**:

|Protocol|Purpose|
|---|---|
|**NetBIOS**|Network Basic Input/Output System; manages sessions|
|**PPTP**|Point-to-Point Tunneling Protocol; creates VPN connections|
|**L2TP**|Layer 2 Tunneling Protocol; tunneling protocol for VPNs|
|**SSH**|Secure Shell; secure remote login and session management|
|**NFS**|Network File System; manages file sharing sessions|

**Key Features**:

- Synchronization checkpoints
- Session recovery
- Duplex communication control

---

## Layer 6: Presentation Layer

Translates, encrypts, and compresses data into a format that both sender and receiver understand.

**Key Functions**: Data encryption, decryption, compression, translation, formatting

**Protocols & Technologies**:

|Type|Examples|Purpose|
|---|---|---|
|**Encryption**|SSL, TLS|Secures data transmission|
|**Compression**|JPEG, MPEG, GIF|Reduces data size|
|**Translation**|ASCII, UNICODE|Converts character encoding|
|**VPN**|IPSec, L2TP|Creates secure tunnels|

**File Formats Handled**:

- Images: JPEG, PNG, GIF
- Video: MPEG, MP4
- Audio: MP3, WAV
- Text: ASCII, Unicode, EBCDIC

---

## Layer 7: Application Layer

Provides network services directly to user applications and enables user interaction with the network.

**Key Functions**: User interface, data formatting, resource sharing, network services

**Protocols**:

|Category|Protocol|Purpose|
|---|---|---|
|**Name System**|DNS|Translates domain names to IP addresses|
|**Email**|SMTP|Sends email between clients and servers|
||POP3|Retrieves and downloads email to client|
||IMAP|Accesses email on server|
|**File Transfer**|FTP|Reliable file transfer|
||SFTP|Secure file transfer (SSH encrypted)|
||TFTP|Fast, lightweight file transfer|
||SMB/CIFS|Network file sharing (Windows)|
|**Web**|HTTP|Transfers web content|
||HTTPS|Encrypted HTTP|
||REST|Web services using APIs|
|**Host Config**|DHCP|Assigns IP addresses dynamically|
||DHCPv4|IPv4 dynamic assignment|
||DHCPv6|IPv6 dynamic assignment|
||SLAAC|IPv6 self-assignment|
|**Remote Access**|Telnet|Remote terminal access (unencrypted)|
||SSH|Secure remote access|
|**Directory Services**|LDAP|Lightweight Directory Access Protocol|
|**Monitoring**|SNMP|Simple Network Management Protocol|
|**Domain**|DNSSEC|Secure DNS|
|**VoIP**|SIP|Session Initiation Protocol|
|**Video Conferencing**|H.323|Multimedia communication protocol|

**Applications**:

- Web browsers
- Email clients
- File transfer utilities
- Remote desktop applications
