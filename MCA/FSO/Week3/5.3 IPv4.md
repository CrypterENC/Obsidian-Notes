
# IPv4 - Simple Guide

## What is IPv4?

IPv4 is **Internet Protocol version 4** - a set of rules that allows computers to send data to each other over networks.

Think of it like a mailing system:

- IPv4 address = Home address
- Router = Post office
- Default gateway = The door to reach other neighborhoods

---

## IPv4 Basic Operations

When two devices communicate using IPv4, these 4 things happen:

|Step|Name|What It Does|
|---|---|---|
|1|**Addressing**|Each device gets a unique IPv4 address|
|2|**Encapsulation**|Data is wrapped with IP header (sender & receiver address)|
|3|**Routing**|Data travels through routers to reach destination|
|4|**De-encapsulation**|Destination device unwraps the data and reads it|

---

## Key Characteristics of IPv4

|Feature|Meaning|
|---|---|
|**Connectionless**|No connection needed before sending data (like mailing a letter)|
|**Best Effort**|Not guaranteed to arrive (IP doesn't track if packet arrives)|
|**Media Independent**|Works over any medium (copper cables, fiber, wireless)|
|**Unreliable**|No error correction or retransmission (TCP handles that)|

## IPv4 Addresses

### What is an IPv4 Address?

An IPv4 address is a **32-bit number** written as 4 octets (groups of 8 bits) separated by dots.

**Example**: `192.168.10.5`

### Network and Host Portions

An IPv4 address has two parts:

```
192.168.10.5
│            └─ HOST PORTION (identifies specific device)
└──────────────── NETWORK PORTION (identifies the network)
```

**Example**:

- Network: `192.168.10.0` (the network)
- Host: `192.168.10.5` (specific device on that network)

All devices on the same network share the same **network portion**.

---

## Subnet Mask

### What is a Subnet Mask?

A subnet mask tells you which part of the IPv4 address is the **network** and which part is the **host**.

### Common Subnet Masks

|Subnet Mask|Prefix|Network Bits|Host Bits|Number of Hosts|
|---|---|---|---|---|
|255.255.255.0|/24|24 bits|8 bits|254|
|255.255.0.0|/16|16 bits|16 bits|65,534|
|255.0.0.0|/8|8 bits|24 bits|16,777,214|

### Example: Understanding Subnet Mask

**Address**: `192.168.10.10` **Subnet Mask**: `255.255.255.0` (or /24)

|Decimal|Binary|
|---|---|
|Address: 192.168.10.10|11000000.10101000.00001010.00001010|
|Mask: 255.255.255.0|11111111.11111111.11111111.00000000|
|**Network**: 192.168.10.0|11000000.10101000.00001010.00000000|

**Result**: Network is `192.168.10.0`, device is `.10` on that network

---

## Prefix Length (Slash Notation)

Instead of writing `/24`, you can write the address with prefix:

**Example**: `192.168.10.10/24`

This means:

- First **24 bits** = network portion
- Remaining **8 bits** = host portion

## Types of IPv4 Addresses

### 1. Class A Addresses

- **Range**: `0.0.0.0` to `127.255.255.255`
- **Prefix**: /8
- **Supports**: Over 16 million hosts
- **Use**: Very large networks
- **Example**: `10.0.0.0/8`

### 2. Class B Addresses

- **Range**: `128.0.0.0` to `191.255.255.255`
- **Prefix**: /16
- **Supports**: About 65,000 hosts
- **Use**: Medium to large networks
- **Example**: `172.16.0.0/16`

### 3. Class C Addresses

- **Range**: `192.0.0.0` to `223.255.255.255`
- **Prefix**: /24
- **Supports**: 254 hosts
- **Use**: Small networks
- **Example**: `192.168.1.0/24`

### 4. Class D Addresses

- **Range**: `224.0.0.0` to `239.255.255.255`
- **Use**: Multicast (sending to multiple devices)

### 5. Class E Addresses

- **Range**: `240.0.0.0` to `255.255.255.255`
- **Use**: Experimental (not used)

## Public vs Private IPv4 Addresses

### Public IPv4 Addresses

- **Globally unique** across the entire internet
- **Routable** - can be reached from anywhere on the internet
- **Assigned by**: ISPs
- **Used for**: Websites, servers, devices accessible on internet

### Private IPv4 Addresses

- **Used internally** within organizations and homes
- **NOT routable** on the internet
- **Can be reused** by different organizations
- **Protected** - internet routers ignore them

### Private Address Ranges

|Block|Range|Number of Addresses|
|---|---|---|
|Class A Private|10.0.0.0 to 10.255.255.255|16 million|
|Class B Private|172.16.0.0 to 172.31.255.255|1 million|
|Class C Private|192.168.0.0 to 192.168.255.255|65,000|

**Examples**:

- Home Wi-Fi devices: `192.168.1.100`
- Office computers: `10.0.0.50`
- School network: `172.16.10.25`

## Network Address Translation (NAT)

### What is NAT?

NAT translates **private addresses** to **public addresses** so internal devices can communicate on the internet.

### How NAT Works

```
Internal Network          Router (NAT)          Internet
┌─────────────────┐      ┌──────────┐      ┌─────────────┐
│ 192.168.1.10    │──┐   │ Private  │──┐   │ ISP Public  │
│ 192.168.1.20    │──┼──▶│ to Public│──┼──▶│ 203.0.113.5 │
│ 192.168.1.30    │──┘   │ Translation   │
└─────────────────┘      └──────────┘      └─────────────┘
```

**Example**:

- Your home device: `192.168.1.10` (private)
- Router converts to: `203.0.113.5` (public)
- Internet sees only: `203.0.113.5`

## Default Gateway

### What is a Default Gateway?

A **default gateway** is the router that connects your local network to other networks and the internet.

**Analogy**: If your network is a room, the default gateway is the **door** to exit the room.

### Key Features of Default Gateway

✓ Has an IP address on the same network as your devices ✓ Forwards traffic to other networks ✓ Routes data to the internet ✓ **Required** to reach remote networks

### Example

```
HOME NETWORK                    INTERNET
┌──────────────────┐           ┌─────────────┐
│ PC1: 192.168.1.10│           │  Google     │
│ PC2: 192.168.1.20│──┐        │  Facebook   │
│ PC3: 192.168.1.30│  │        │  Other Sites│
└──────────────────┘  │        └─────────────┘
                      │
                    ┌─▼────────────┐
                    │ Router       │
                    │ (Gateway)    │
                    │ 192.168.1.1  │
                    └──────────────┘
```

**Default Gateway for all PCs**: `192.168.1.1`

### How Default Gateway Works

**Scenario 1: Local Communication**

```
PC1 (192.168.1.10) ──[Direct]──▶ PC2 (192.168.1.20)
(No gateway needed - same network)
```

**Scenario 2: Remote Communication**

```
PC1 (192.168.1.10) ──[To Gateway]──▶ Router ──[To Internet]──▶ Google
```

### Configuring Default Gateway

Devices get default gateway address in two ways:

1. **Automatically (DHCP)**
    - Router automatically assigns the default gateway
    - Most common in home/office networks
2. **Manually (Static)**
    - System administrator assigns it manually
    - Used in servers and critical systems

```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Address: 192.168.1.10
  Subnet Mask: 255.255.255.0
  Default Gateway: 192.168.1.1
  Network: 192.168.1.0

Phone:
  Address: 192.168.1.20
  Subnet Mask: 255.255.255.0
  Default Gateway: 192.168.1.1
  Network: 192.168.1.0

Router (Default Gateway):
  Address: 192.168.1.1
  Public IP: 203.0.113.5 (from ISP)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

To access local file on Phone:
  Laptop (192.168.1.10) ──[Direct]──▶ Phone (192.168.1.20)
  Same network, no gateway needed

To access Google.com:
  Laptop (192.168.1.10) ──[To Gateway]──▶ Router ──[Public IP 203.0.113.5]──▶ Internet
  Gateway needed to reach outside network
```
