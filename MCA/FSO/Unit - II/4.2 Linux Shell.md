
## The Linux Shell

- In Linux the **~={yellow}user communicates with the OS=~** by using **~={green}CLI=~**  or **~={green}GUI=~**
- By default, Linux starts in GUI.

### Accessing CLI from GUI:
- Use terminal ~={cyan}**emulator applications**=~
- Common emulators :
	- Terminal
	- term
	- console
	
### Basic Commands

- `mv` - Move or rename files
- `clear` - Clear terminal display
- `cat` - Display file contents
- `chmod` - modifies the file permissions
- `chown` - changes the ownership of a file

### File and Directory Commands

- `ls` - List directories (most frequently used)
- `pwd` - Print working directory
- `cd` - Navigate through directories
- `mkdir` - Create directories
- `cat` - list the contents of a file.
- `grep` - searches for specific strings in characters within a file.

### Working with Text Files

- Linux has many text editors, **~={green}with various features and functions=~** 
- Types:
	- ~={yellow}**Graphical interfaces (convenient, easy)**=~
	- ~={yellow}**Command-line only (important for remote access)=~**
- Some text editors focus on the programmer and include features such as :
	- ~={yellow}**Syntax highlight**=~
	- ~={yellow}**Bracket checking**=~

### The importance of text files in Linux

#### "Everything is a File" in Linux :

- **Memory treated as file**
- **Disks treated as file**
- **Monitor treated as file**
- **Directories treated as file**

#### Configuration Files :

- These files are used to ~={yellow}**store adjustment and settings**=~ **~={green}for specific applications or services=~**

#### User Modification :

- ~={yellow}**Users with proper permission levels**=~ can use text editors to ~={green}**change the contents of configuration files**=~.

**Example:**

- Command: `sudo nano /etc/hosts`
- `sudo` = superuser do (invoke superuser privilege)
- `nano` = text editor
- `/etc/hosts` = host file
