# Scripting and Automation in Python

## Scripting Basics
- **Running scripts**: `python script.py` or `./script.py` (with shebang `#!/usr/bin/env python3`).
- **Command-line arguments**: Use `sys.argv` or `argparse` for inputs.
- **Exit codes**: Use `sys.exit(code)` to indicate success/failure.

## Key Modules for Automation
- **os**: Interact with OS (files, directories, environment).
- **subprocess**: Run external commands.
- **shutil**: High-level file operations (copy, move, remove).
- **schedule**: Schedule tasks (e.g., `schedule.every().day.at("10:00").do(job)`).
- **time**: Add delays or get timestamps.

## Examples
### File Backup Automation
```python
import shutil
import os

def backup_files(src, dst):
    if os.path.exists(src):
        shutil.copytree(src, dst, dirs_exist_ok=True)
        print("Backup complete")

backup_files('/source', '/backup')
```

### Running System Commands
```python
import subprocess

result = subprocess.run(['ls', '-l'], capture_output=True, text=True)
print(result.stdout)
```

### Scheduled Task
```python
import schedule
import time

def job():
    print("Task executed")

schedule.every(10).seconds.do(job)

while True:
    schedule.run_pending()
    time.sleep(1)
```

## Best Practices
- Handle errors with try-except.
- Use logging for automation scripts.
- Test scripts in small parts.
- Secure sensitive data (e.g., API keys).
