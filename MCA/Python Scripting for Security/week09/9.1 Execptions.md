# Exception Handling in Python

## üîç Basic Structure

```python
try:
    # Code that may raise an exception
    risky_operation()
except SomeException as e:
    # Handle the exception
    print(f"Error: {e}")
else:
    # Executes only if no exception occurs
    print("Success!")
finally:
    # Always executes, regardless of exceptions
    print("Cleanup done.")
```

## üìã Components Explained

### Try Block
- **Purpose**: Contains code that might raise exceptions.
- **When used**: Wrap risky operations here.

### Except Block
- **Purpose**: Catches and handles specific exceptions.
- **Multiple except**: Can have multiple for different exception types.
- **Syntax**: `except ExceptionType as variable:`

### Else Block
- **Purpose**: Runs only if no exception in try block.
- **Use case**: Code that should run on success.

### Finally Block
- **Purpose**: Always executes, for cleanup tasks.
- **Examples**: Closing files, releasing resources.

## üö® Common Built-in Exceptions

| Exception             | Description                     | Example Trigger             |
| --------------------- | ------------------------------- | --------------------------- |
| **ValueError**        | Invalid value for operation     | `int("abc")`                |
| **TypeError**         | Operation on incompatible types | `"a" + 1`                   |
| **ZeroDivisionError** | Division by zero                | `10 / 0`                    |
| **IndexError**        | List index out of range         | `lst[10]` on short list     |
| **KeyError**          | Dictionary key not found        | `d["missing_key"]`          |
| **FileNotFoundError** | File not found                  | `open("nonexistent.txt")`   |
| **IOError**           | Input/Output operation fails    | Disk full during write      |
| **AssertionError**    | Assert statement failure        | `assert False`              |
| **AttributeError**    | Attribute not found on object   | `obj.nonexistent_attr`      |
| **NameError**         | Variable not defined            | `print(undefined_var)`      |
| **ImportError**       | Module not found                | `import nonexistent_module` |
| **SyntaxError**       | Invalid Python syntax           | `if True print("hi")`       |
| **IndentationError**  | Incorrect indentation           | Mixed tabs/spaces           |

## üí° Practical Example

```python
def safe_divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("‚ùå Cannot divide by zero!")
        return None
    except TypeError:
        print("‚ùå Invalid types for division!")
        return None
    else:
        print("‚úÖ Division successful!")
        return result
    finally:
        print("üßπ Operation completed.")

# Test it
safe_divide(10, 2)  # Success
safe_divide(10, 0)  # ZeroDivisionError
```

### AssertionError Example
```python
def check_positive(num):
    assert num > 0, "Number must be positive!"
    return num * 2

try:
    check_positive(-5)
except AssertionError as e:
    print(f"‚ùå Assertion failed: {e}")
```

**How assert works**: The `assert` statement checks a condition. If the condition is `True`, nothing happens. If `False`, it raises an `AssertionError` with an optional message. Useful for debugging and enforcing invariants.

## ‚úÖ Best Practices
- **Be specific**: Catch specific exceptions, not bare `except:`.
- **Use finally**: For resource cleanup.
- **Log errors**: Print or log exception details for debugging.
- **Reraise if needed**: Use `raise` to re-raise exceptions after handling.
