# Requests Module in Python

## Installation
- Install via pip: `pip install requests`
- Popular HTTP library for Python

## Basic Usage
```python
import requests

# GET request
response = requests.get('https://api.example.com/data')

# POST request
data = {'key': 'value'}
response = requests.post('https://api.example.com/data', json=data)
```

## Response Object Attributes and Methods
### Attributes
- `response.status_code`: HTTP status code (e.g., 200 for success)
- `response.text`: Response content as a string (decoded)
- `response.content`: Response content as raw bytes
- `response.json()`: Parse response as JSON (method)
- `response.headers`: Dictionary of response headers
- `response.url`: Final URL after redirects
- `response.encoding`: Text encoding (e.g., 'utf-8')
- `response.apparent_encoding`: Apparent encoding from content
- `response.cookies`: CookieJar object with response cookies
- `response.elapsed`: Time elapsed for the request (timedelta)
- `response.history`: List of response objects from redirects
- `response.is_permanent_redirect`: True if permanent redirect
- `response.is_redirect`: True if redirect
- `response.links`: Dictionary of parsed link headers
- `response.next`: Next request in a redirect chain
- `response.ok`: True if status_code is less than 400
- `response.reason`: Textual reason for status code (e.g., 'OK')
- `response.request`: PreparedRequest object that was sent
- `response.raw`: Underlying urllib3 response object

### Methods
- `response.raise_for_status()`: Raise exception if status is not 2xx
- `response.close()`: Close the connection
- `response.iter_content(chunk_size=1, decode_unicode=False)`: Iterate over response content in chunks
- `response.iter_lines(chunk_size=512, decode_unicode=None, delimiter=None)`: Iterate over lines in response

## Advanced Features
- **Parameters**: `params={'key': 'value'}` for query strings
- **Headers**: `headers={'User-Agent': 'MyApp'}`
- **Authentication**: `auth=('username', 'password')`
- **Timeouts**: `timeout=5` (seconds)
- **Sessions**: Reuse connections with `requests.Session()`

## Notes
- Handle exceptions: `requests.exceptions.RequestException`
- Check status: `response.raise_for_status()` raises for bad status
- Use sessions for multiple requests to same host
